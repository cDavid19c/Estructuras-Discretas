Índice de algoritmos

01

Coeficientes binomiales con programación dinámica

Calcula C(n, k) (el número de formas de elegir k elementos de n) reutilizando subproblemas mediante el Triángulo de Pascal: C(n,k) = C(n-1,k-1) + C(n-1,k).

Cómo se ejecuta

  1. Se construye una tabla donde la fila i contiene los coeficientes de (a+b)i.
  2. Bordes en 1: C(i,0)=C(i,i)=1.
  3. Cada celda interior suma las dos celdas superiores.
  4. Se lee la celda (n, k).

Complejidad

Tiempo O(n·k)Espacio O(n²) tabla
02

Paquetes de red — calcular P(A) analíticamente y estimar con simulación

Se envían n paquetes; cada uno llega de forma independiente con probabilidad p. El evento A = «llegan al menos k paquetes». Se compara la probabilidad exacta (suma binomial) con una estimación de Monte Carlo.

Cómo se ejecuta

  1. Analítica: P(A)=Σi=kn C(n,i)·pi·(1-p)n-i.
  2. Simulación: en cada ensayo se sortean n paquetes y se cuenta si llegan ≥ k.
  3. La estimación es (ensayos exitosos)/(total de ensayos).
  4. Se reporta el error absoluto entre ambas.

Complejidad

Analítica O(n)Simulación O(T·n)
03

Redundancia paralela

Varios componentes en paralelo: el sistema funciona si al menos uno funciona. La confiabilidad total es R = 1 − ∏(1 − ri), es decir, 1 menos la probabilidad de que todos fallen.

Cómo se ejecuta

  1. Se ingresan las confiabilidades ri de cada componente (0 a 1).
  2. Se multiplica la probabilidad de falla (1 − ri) de cada uno.
  3. R = 1 − ese producto.
  4. Más componentes ⇒ mayor confiabilidad.

Complejidad

Tiempo O(n)Espacio O(1)
04

Ley de la Probabilidad Total + Bayes básico

Dadas causas mutuamente excluyentes Bi con probabilidad previa y la verosimilitud P(A|Bi), se calcula P(A) por la ley total y luego la probabilidad posterior P(Bi|A) por Bayes.

Cómo se ejecuta

  1. Cada línea es «previa, verosimilitud» de una causa.
  2. Ley total: P(A)=Σ P(Bi)·P(A|Bi).
  3. Bayes: P(Bi|A)=P(Bi)·P(A|Bi)/P(A).
  4. Las previas deberían sumar 1.

Complejidad

Tiempo O(n)
Ejemplo: enfermedad rara (previa 1%) con prueba 99% sensible y 5% de falsos positivos. La fila 1 es «enfermo», la fila 2 «sano».
05

Naive Bayes diagnóstico

Clasificador que asume independencia condicional entre síntomas. Para cada clase calcula P(clase)·∏ P(síntoma|clase), luego normaliza para obtener la probabilidad de cada diagnóstico.

Cómo se ejecuta

  1. Marca los síntomas observados en el paciente.
  2. Por cada clase: prior × producto de verosimilitudes (presente o ausente).
  3. Se normalizan los puntajes para que sumen 1.
  4. Se elige la clase con mayor probabilidad.

Complejidad

Tiempo O(clases·síntomas)
Tabla de verosimilitudes P(síntoma=sí | clase). Clases: Gripe vs. Resfriado.
06

Colisiones en tabla hash

Inserta claves en una tabla de tamaño m usando h(k)=k mod m y encadenamiento (listas por casilla). Cuenta las colisiones y muestra el factor de carga.

Cómo se ejecuta

  1. Por cada clave se calcula su casilla con k mod m.
  2. Si la casilla ya tenía elementos, se cuenta una colisión.
  3. La clave se agrega a la cadena de esa casilla.
  4. Factor de carga α = n/m.

Complejidad

Inserción prom. O(1)Peor caso O(n)
07

Combinaciones con repetición — O(k)

Cuenta las formas de elegir k elementos de n tipos permitiendo repetición: C(n+k−1, k). Se calcula con la fórmula multiplicativa en tiempo O(k), sin construir tablas.

Cómo se ejecuta

  1. Resultado = 1.
  2. Para i = 0 … k−1: resultado ×= (n+i)/(i+1).
  3. Cada paso mantiene un entero exacto.
  4. El total es C(n+k−1, k).

Complejidad

Tiempo O(k)Espacio O(1)
08

MergeSort + verificación

Ordena dividiendo el arreglo a la mitad, ordenando cada mitad recursivamente y combinando. Tras ordenar, verifica que el resultado quede no decreciente.

Cómo se ejecuta

  1. Dividir: partir el arreglo por la mitad.
  2. Conquistar: ordenar cada mitad recursivamente.
  3. Combinar: mezclar las dos mitades ordenadas.
  4. Verificar: comprobar a[i] ≤ a[i+1] para todo i.

Complejidad

Tiempo O(n log n)Espacio O(n)
09

Árbol de recursión + verificación del Teorema Maestro

Para recurrencias T(n)=a·T(n/b)+Θ(nd) determina el caso del Teorema Maestro y la complejidad. Despliega el árbol de recursión (trabajo por nivel) como verificación.

Cómo se ejecuta

  1. Comparar d con logb(a).
  2. d < logba ⇒ Caso 1: Θ(nlogba).
  3. d = logba ⇒ Caso 2: Θ(nd log n).
  4. d > logba ⇒ Caso 3: Θ(nd).

Complejidad

Resuelve la recurrencia
MergeSort: a=2, b=2, d=1 ⇒ Caso 2 ⇒ Θ(n log n).
10

Lista de adyacencia y matriz de adyacencia — operaciones O(·)

Construye un grafo en ambas representaciones y compara el costo de las operaciones fundamentales. Puedes probar si existe una arista (u, v) en cada estructura.

Cómo se ejecuta

  1. Matriz: M[u][v]=1 si hay arista (acceso directo).
  2. Lista: cada vértice guarda sus vecinos.
  3. Matriz favorece consultas; lista favorece grafos dispersos.
  4. Prueba la consulta de arista u–v.

Complejidad

Matriz: espacio O(V²)Lista: espacio O(V+E)
11

Calcular grados desde lista de adyacencia

Recorre la lista de adyacencia para contar el grado de cada vértice y verifica el lema del apretón de manos: Σ grados = 2·|aristas|.

Cómo se ejecuta

  1. El grado de v = tamaño de su lista de vecinos.
  2. Se suman todos los grados.
  3. La suma debe ser el doble del número de aristas.

Complejidad

Tiempo O(V+E)
12

Verificar isomorfismo por invariantes

Compara dos grafos según invariantes necesarios: número de vértices, número de aristas y secuencia de grados ordenada. Si difieren, no son isomorfos; si coinciden, son candidatos.

Cómo se ejecuta

  1. Calcular |V|, |E| y la secuencia de grados de cada grafo.
  2. Si algún invariante difiere ⇒ NO isomorfos.
  3. Si todos coinciden ⇒ posiblemente isomorfos.
  4. Los invariantes son necesarios, no suficientes.

Complejidad

Tiempo O(V+E)
13

Conectividad y circuito de Euler

Verifica si el grafo es conexo y clasifica su recorrido de Euler: hay circuito si todos los vértices tienen grado par; hay camino si exactamente dos tienen grado impar.

Cómo se ejecuta

  1. Conectividad: BFS desde un vértice no aislado.
  2. Contar vértices de grado impar.
  3. 0 impares + conexo ⇒ circuito de Euler.
  4. 2 impares + conexo ⇒ camino de Euler.

Complejidad

Tiempo O(V+E)
14

BFS — Búsqueda en Anchura (Breadth-First Search)

Explora el grafo por niveles desde un origen usando una cola. Encuentra el orden de visita y la distancia mínima (en aristas) a cada vértice.

Cómo se ejecuta

  1. Encolar el origen con distancia 0.
  2. Sacar de la cola, visitar y encolar vecinos no visitados.
  3. La distancia del vecino = distancia actual + 1.
  4. Termina cuando la cola se vacía.

Complejidad

Tiempo O(V+E)
15

DFS — Búsqueda en Profundidad (Depth-First Search)

Explora tan profundo como sea posible antes de retroceder, usando recursión (pila). Devuelve el orden de descubrimiento de los vértices.

Cómo se ejecuta

  1. Marcar el origen como visitado.
  2. Para cada vecino no visitado, llamar DFS recursivamente.
  3. Al no quedar vecinos, retroceder.
  4. Se registra el orden en que se descubren.

Complejidad

Tiempo O(V+E)
16

Algoritmo de Prim

Construye el árbol de expansión mínima (MST) haciendo crecer un único árbol: en cada paso añade la arista más barata que conecta un vértice del árbol con uno fuera de él.

Cómo se ejecuta

  1. Comenzar con un vértice cualquiera.
  2. Elegir la arista mínima que salga del árbol actual.
  3. Agregar el nuevo vértice al árbol.
  4. Repetir hasta incluir todos los vértices.

Complejidad

O(V²) o O(E log V)
17

Kruskal con Union-Find

Construye el MST ordenando las aristas por peso y añadiéndolas si no forman ciclo. La detección de ciclos usa Union-Find (conjuntos disjuntos) con compresión de caminos.

Cómo se ejecuta

  1. Ordenar todas las aristas por peso ascendente.
  2. Cada vértice empieza en su propio conjunto.
  3. Tomar una arista si une dos conjuntos distintos.
  4. Unir esos conjuntos; descartar si crearían ciclo.

Complejidad

O(E log E)
18

Verificar árbol + contar hojas y altura

Comprueba si un grafo no dirigido es un árbol (conexo y con exactamente |V|−1 aristas, sin ciclos). De serlo, cuenta las hojas y calcula la altura desde una raíz dada.

Cómo se ejecuta

  1. Árbol ⇔ conexo y |E| = |V| − 1.
  2. Hojas: vértices de grado 1.
  3. Altura: BFS desde la raíz, mayor nivel alcanzado.

Complejidad

Tiempo O(V+E)
19

Recorridos preorden e inorden

Construye un árbol binario de búsqueda con los valores dados y muestra el recorrido preorden (raíz, izq, der) e inorden (izq, raíz, der). El inorden de un BST queda ordenado.

Cómo se ejecuta

  1. Insertar los valores en un BST.
  2. Preorden: visitar raíz, luego subárbol izquierdo, luego derecho.
  3. Inorden: izquierdo, raíz, derecho.
  4. El inorden produce los valores en orden creciente.

Complejidad

Recorrido O(n)
20

Recorrido postorden y evaluación de AST

Analiza una expresión aritmética en un árbol de sintaxis (AST), muestra su recorrido postorden (notación polaca inversa) y la evalúa de abajo hacia arriba.

Cómo se ejecuta

  1. Parsear la expresión respetando precedencia y paréntesis.
  2. Construir el AST con operadores y operandos.
  3. Postorden: izquierdo, derecho, raíz ⇒ RPN.
  4. Evaluar combinando los hijos en cada nodo.

Complejidad

Tiempo O(n)
Operadores soportados: + − * / y paréntesis.
21

Recorrido por niveles (Level-Order / BFS en árbol)

Visita el árbol nivel por nivel usando una cola. Es BFS aplicado a un árbol: primero la raíz, luego sus hijos, después los nietos, etc.

Cómo se ejecuta

  1. Encolar la raíz.
  2. Sacar un nodo, visitarlo y encolar sus hijos.
  3. Repetir hasta vaciar la cola.
  4. Se reporta cada nivel por separado.

Complejidad

Tiempo O(n)
22

Búsqueda en BST

Busca un valor en un árbol binario de búsqueda comparando en cada nodo: si es menor va a la izquierda, si es mayor a la derecha. Se resalta el camino recorrido.

Cómo se ejecuta

  1. Comparar el objetivo con la raíz.
  2. Menor ⇒ ir al hijo izquierdo; mayor ⇒ al derecho.
  3. Igual ⇒ encontrado.
  4. Llegar a vacío ⇒ no está.

Complejidad

Prom. O(log n)Peor caso O(n)
23

Inserción en BST

Inserta valores manteniendo la propiedad del BST. Cada nuevo valor desciende comparando hasta encontrar el lugar vacío correcto. El árbol se actualiza en vivo.

Cómo se ejecuta

  1. Desde la raíz, comparar el nuevo valor.
  2. Menor ⇒ bajar a la izquierda; mayor ⇒ a la derecha.
  3. Al llegar a un lugar vacío, colocar el nodo.
  4. Los duplicados se ignoran.

Complejidad

Prom. O(log n)
Árbol inicial: 50, 30, 70, 20, 40, 60, 80.
24

Eliminación en BST

Elimina un valor manejando los tres casos: hoja (se borra), un hijo (se reemplaza por él) y dos hijos (se reemplaza por el sucesor inorden, el mínimo del subárbol derecho).

Cómo se ejecuta

  1. Localizar el nodo a eliminar.
  2. Hoja ⇒ quitarlo directamente.
  3. Un hijo ⇒ el hijo ocupa su lugar.
  4. Dos hijos ⇒ sustituir por el sucesor inorden.

Complejidad

Prom. O(log n)
Árbol inicial: 50, 30, 70, 20, 40, 60, 80.
25

Rotaciones AVL — Los 4 tipos

Árbol AVL que se autobalancea al insertar. Cuando el factor de balance se rompe aplica una de las cuatro rotaciones: LL (derecha), RR (izquierda), LR (izq-der) o RL (der-izq).

Cómo se ejecuta

  1. Insertar como en un BST y recalcular alturas.
  2. Factor de balance = altura(izq) − altura(der).
  3. |FB| > 1 ⇒ desbalance: aplicar rotación.
  4. LL→rotar derecha, RR→izquierda, LR y RL→dobles.

Complejidad

Inserción O(log n)Rotación O(1)
